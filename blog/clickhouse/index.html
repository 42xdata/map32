ClickHouse Column-Oriented DBMS

2016-06-15

Yandex release a new distributed column-oriented DBMS today called ClickHouse. Well, Yandex has a Top Level Domain.

While the homepage is fancier, the reference document is much more informative.

Here is the claimed distinctive features of Clickhouse: More preciesly, they are the distinctive features compared with a row-oriented DB.

1. True column-oriented DBMS.
2. Data compression.
3. Disk storage of data.
4. Parallel processing on multiple cores.
5. Distributed processing on multiple servers.
6. SQL support.
7. Vector engine.
8. Real-time data updates.
9. Indexes.
10. Suitable for online queries.
11. Support for approximated calculations.
12. Support for nested data structures. Support for arrays as data types.
13. Support for restrictions on query complexity, along with quotas.
14. Data replication and support for data integrity on replicas.

It is well known that the column-oriented DBMS family is the king of OLAP workload. From the academeic point of a view, these points are not very technical. For example, #2 Data Compression is a standard feature of a C* DB. You may find the same list in any modern column-oriened database nowadays. But it is always to glad to see more open source projects that serves real world production.

I can also imagine to support nested data types (#12), it may invent another SQL dialect.

I am looking forward to read the design along with the suvery paper of the column-store.
